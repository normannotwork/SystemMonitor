# 🚀 SysMon - Утилита мониторинга системы в реальном времени

**SysMon** — это легковесный, кроссплатформенный (Linux) сервис для мониторинга состояния системы в режиме реального времени.


## 📝 Зачем и как это работает?
- **Бэкенд (C++17):** Запускает собственный многопоточный HTTP-сервер. В фоновом потоке собирает метрики CPU, RAM, сети и Load Average.
- **Фронтенд (HTML/CSS/TS):** Подключается к серверу, получает поток данных в формате JSON и динамически отрисовывает интерфейс без перезагрузки страницы.

## 🛠 Технологии и системные требования

Для успешной сборки и запуска проекта на вашем компьютере должны быть установлены следующие инструменты:

1. **C++ Compiler (GCC/Clang)** с поддержкой **C++17**.
2. **CMake** (версия 3.10 или выше) и **Make** — для конфигурации и сборки бэкенда.
3. **Node.js** (версия 14.x или выше) и **npm** — для компиляции TypeScript кода фронтенда.
4. **ОС:** Linux (так как используются системные файлы `/proc/stat`, `/proc/meminfo` и т.д.).

## 🚀 Запуск и сборка

Вы можете собрать проект двумя способами: запустить готовый автоматический скрипт (рекомендуется) или выполнить сборку вручную шаг за шагом, чтобы понять, как всё устроено под капотом.

### Способ 1: Автоматическая сборка (в 1 клик)

Самый быстрый способ. Скрипт `build.sh` сам установит NPM-пакеты, соберет фронтенд, разложит файлы по папкам и скомпилирует C++ сервер.

1. Дайте скрипту права на исполнение:
   ```bash
   chmod +x build.sh
   ```
2. Запустите скрипт:
   ```bash
   ./build.sh
   ```
3. После успешной сборки запустите сервер:
   ```bash
   ./build/sysmon
   ```

---

### Способ 2: Ручная (пошаговая) сборка

Если вы хотите полностью контролировать процесс или у вас возникли проблемы со скриптом, выполните эти шаги вручную:

#### Шаг 1. Сборка Фронтенда
Серверу для раздачи нужны готовые статические файлы. Нам нужно скомпилировать TypeScript в JavaScript и положить всё в папку `dist`.

```bash
# Переходим в директорию фронтенда
cd frontend

# Устанавливаем зависимости из package.json
npm install

# Компилируем TypeScript код
npm run build

# Создаем папку для готовых файлов (если её нет)
mkdir -p dist

# Копируем HTML и CSS в папку раздачи
cp src/index.html src/style.css dist/

# Переименовываем скомпилированный JS файл, чтобы сервер мог его найти
mv dist/main.js dist/app.js 2>/dev/null || true

# Возвращаемся в корень проекта
cd ..
```

#### Шаг 2. Сборка Бэкенда (C++)
Теперь собираем серверную часть с помощью CMake.

```bash
# Создаем директорию для сборки, чтобы не засорять корень исходников
mkdir -p build
cd build

# Генерируем Makefile с помощью CMake
cmake ..

# Компилируем проект, используя все доступные ядра процессора для ускорения
make -j$(nproc)

# Возвращаемся в корень
cd ..
```

## 🖥 Как использовать

После того как проект собран любым из способов, сервер готов к работе.

1. Запустите исполняемый файл из корневой директории:
   ```bash
   ./build/sysmon
   ```
2. Откройте ваш любимый веб-браузер и перейдите по адресу:
   ```text
   http://localhost:8080
   ```
   *(Примечание: точный порт будет выведен в консоли при запуске сервера)*

3. Чтобы остановить сервер, нажмите `Ctrl+C` в терминале. Сервер корректно обработает сигнал, закроет порты и завершит потоки.
